# Multi-Agent Architectures for Scientific Discovery

> **Pre-print scope**: The AI Co-Scientist (arXiv:2502.18864), EvoScientist, Indibator,
> BioDisco, and DASES are all arXiv pre-prints not yet peer-reviewed. The sycophancy
> finding from Perez et al. (arXiv:2212.09251) appeared in ACL 2023 findings - peer-reviewed.

## Overview

Science is a social enterprise. The individual scientist's cognitive limitations
are compensated by peer review, replication, adversarial debate, and collective
synthesis. AI agents doing science in isolation reproduce the worst properties
of the lone researcher with no oversight.

Multi-agent architectures are the computational implementation of science's
social epistemology: multiple agents with distinct roles, genuine disagreement,
and adversarial critique produce more reliable outputs than single agents.

This document covers the design principles and current state-of-the-art
multi-agent systems for scientific discovery.

---

## Why Single-Agent Science Fails

### The Echo Chamber Problem

A single LLM agent that generates a hypothesis and then evaluates it with the
same model has not obtained independent review. The generator and critic share
the same weights, the same training distribution, the same systematic biases.

This is the computational analog of a researcher who:
- Designs their own study
- Collects their own data
- Analyzes their own results
- Writes their own peer review

Science evolved peer review precisely to break this self-confirming loop.

### The Sycophancy Problem

LLMs are trained with RLHF on human preference data. Humans prefer confident,
coherent, agreeable responses. This creates a sycophancy bias: given its own
prior output, an LLM tends to generate text that confirms rather than
challenges it.

An LLM evaluating its own scientific claim is likely to find it compelling - 
not because the claim is correct but because it was generated by a confident,
coherent process that the same model associates with quality.

Source: Perez et al. (2022). Discovering Language Model Behaviors with Model-Written
Evaluations. arXiv:2212.09251. (Sycophancy is one of several inverse-scaling behaviors
documented in this paper.)

## The Role Architecture of Scientific Peer Review

The multi-agent systems below all map to roles in human scientific practice.
The mapping is not accidental - it reflects what peer review is actually for:

| Human role | Function | AI agent equivalent |
|---|---|---|
| Researcher | Generates hypotheses, designs experiments | Generator agent |
| Lab member | Executes experiments, collects data | Experimenter / Engineer agent |
| Reviewer 1 | Checks methodology, statistical validity | Critic agent |
| Reviewer 2 | Checks novelty, grounding in literature | Reflection / Proximity agent |
| Editor | Synthesizes reviews, decides acceptance | Meta-review / Ranking agent |
| Replicator | Independently repeats the experiment | Validator agent |

The best current systems implement 3–6 of these roles. None have implemented all.

## AI Co-Scientist (Google Research, 2025)

**Paper**: arXiv:2502.18864 (2025)
*Towards an AI Co-Scientist*

This is the most architecturally complete multi-agent scientific system published.
Google Research's system (not DeepMind; published Feb 2025 from Google's core
research division) uses a **tournament-based evolution** approach.

### Agents

```
Generation Agent     - Proposes novel hypotheses from literature synthesis
Reflection Agent     - Evaluates each hypothesis for internal consistency,
                       logical validity, and grounding
Ranking Agent        - Compares hypotheses head-to-head; maintains ranked list
Evolution Agent      - Combines and mutates top-ranked hypotheses to generate
                       improved variants (like genetic algorithms)
Proximity Agent      - Checks novelty: how different is this from existing work?
Meta-review Agent    - Synthesizes across the full pipeline; final quality gate
```

### Tournament Process

Rather than evaluating hypotheses in isolation, the system runs them in
head-to-head comparisons (tournament-style), updating rankings based on
pairwise outcomes. This implements a structured debate: hypotheses must
survive comparison with competitors, not just pass a threshold.

### Results

- Validated in biomedical research: drug repurposing predictions confirmed
  in laboratory validation
- Novel target discovery: proposed new biological targets not in training data
- Capsid protein engineering: AI-generated designs experimentally validated

**Why it works**: The tournament structure creates competitive pressure on
hypotheses - survival requires being better than alternatives, not just
passing a static quality bar.

Source: arXiv:2502.18864 (2025).

## EvoScientist - Persistent Memory and Strategy Evolution

**Paper**: arXiv:2603.08127 (2026)
*EvoScientist: Towards Multi-Agent Evolving AI Scientists for End-to-End
Scientific Discovery*

### Key Innovation: Persistent Strategy Memory

Most multi-agent systems are memoryless - each research cycle starts fresh.
EvoScientist introduces **persistent memory modules** that accumulate
strategic knowledge across research cycles:

```
Research Cycle 1:
  Researcher → hypothesis A
  Engineer → experiment on A → result
  Evolution Manager → distills: "in domain D, approach X is more productive
                      than approach Y"
  Stored in: Ideation Strategy Memory, Experimentation Strategy Memory

Research Cycle 2:
  Researcher reads Ideation Strategy Memory → generates better hypothesis B
  (informed by what worked and failed in Cycle 1)
```

**Why this matters**: Human scientists learn from their research experience
over time. A researcher's 10th experiment is better designed than their 1st
because they have accumulated strategic knowledge. EvoScientist operationalizes
this for AI agents.

### Agents

- **Researcher Agent**: Idea generation, literature synthesis, hypothesis
  formulation - reads from Ideation Strategy Memory
- **Engineer Agent**: Experiment execution, code writing, data collection - 
  reads from Experimentation Strategy Memory
- **Evolution Manager Agent**: Distills insights from completed cycles into
  both memory modules

Source: arXiv:2603.08127 (2026).

## Indibator - Individualized Scientist Profiles in Debate

**Paper**: arXiv:2602.01815 (2026)
*Indibator: Diverse and Fact-Grounded Individuality for Multi-Agent Debate
in Molecular Discovery*

### The Generic Persona Problem

Most multi-agent debate systems assign agents to generic roles (Agent 1, Agent 2)
or generic personas (Expert, Skeptic). These agents draw from the same model
with the same weights, producing superficially differentiated but structurally
identical views. The "debate" is theater.

Indibator addresses this with **individualized scientist profiles** derived
from real publication and molecular histories:

```
Agent profile = {
  publication_history: [list of papers by this scientist profile],
  molecular_history:   [compounds this profile has worked with],
  domain_expertise:    [specific sub-domain],
  known_biases:        [systematic tendencies of this scientific community]
}
```

Agents are grounded in distinct factual histories, producing genuinely
differentiated proposals and critiques.

### Debate Structure

```
Proposal phase:   Each agent proposes a molecular candidate based on its profile
Critique phase:   Each agent critiques the others' proposals from its specific
                  perspective
Voting phase:     Agents vote, weighted by the quality of their critique
Output:           Most robust candidate after full debate
```

**Result**: State-of-the-art performance in differentiating agent behavior in
molecular discovery benchmarks. The system produces genuinely diverse proposals
rather than convergent variations on the same theme.

Source: arXiv:2602.01815 (2026).

## BioDisco - Dual-Mode Evidence and Temporal Evaluation

**Paper**: arXiv:2508.01285 (2025)
*BioDisco: Multi-agent hypothesis generation with dual-mode evidence,
iterative feedback and temporal evaluation*

### Dual-Mode Evidence

BioDisco integrates two fundamentally different types of evidence:

1. **Knowledge graph evidence**: Structured, explicit relationships between
   biological entities (protein A interacts with protein B, pathway C activates
   gene D). High precision, limited coverage.

2. **Literature evidence**: Unstructured text from papers. High coverage,
   lower precision.

The two modes are complementary: the knowledge graph provides the skeleton;
the literature fills in gaps and provides context. No prior system integrated
both systematically.

### Temporal Evaluation

Scientific claims have temporal structure: a hypothesis valid in 2015 may be
superseded in 2023. BioDisco implements **temporal evaluation** of retrieved
evidence - how recent is this evidence? How has the field's view evolved?

This prevents agents from reasoning from outdated findings without realizing
they are outdated.

Source: arXiv:2508.01285 (2025).

## DASES - Adaptive Falsification Against Gaming

**Paper**: arXiv:2603.29045 (2026)
*Let the Abyss Stare Back: Adaptive Falsification for Autonomous Scientific
Discovery*

### The Evaluation Gaming Problem

A persistent challenge: AI agents optimize for their evaluation metric.
If the evaluation metric is fixed and static, the agent learns to produce
outputs that score well on the metric without necessarily doing science.
This is the Goodhart's law problem applied to autonomous science.

DASES introduces an **Abyss Falsifier** - a dedicated adversarial agent whose
sole function is to actively probe scientific outputs for failure modes:

```
Scientific agent produces: artifact A (hypothesis, experiment design, result)
Abyss Falsifier:
  - Designs targeted falsification tests for A
  - Tests must be non-obvious (cannot be tests the agent could anticipate)
  - Adaptively evolves test strategy based on what A "defends" successfully
  - At failure: returns detailed failure analysis
  - Acceptance: only if A survives adversarial falsification attempts
```

**Key property**: The falsifier must be genuinely adversarial - structurally
motivated to find flaws, not to confirm quality. This requires separate model
weights and a loss function that rewards finding flaws.

Source: arXiv:2603.29045 (2026).

## Synthesis: Design Principles for Multi-Agent Scientific Systems

| Principle | Implementation | Source |
|---|---|---|
| Genuine role separation | Different weights or adversarial objectives, not persona switching | DASES, Indibator |
| Tournament evaluation | Pairwise comparison vs. isolated threshold testing | AI Co-Scientist |
| Persistent strategic memory | Cross-cycle learning from what worked | EvoScientist |
| Grounded individuation | Agent profiles from real knowledge sources | Indibator |
| Adversarial falsification | Dedicated falsifier agent with anti-confirmation objective | DASES, POPPER |
| Dual-mode evidence | Knowledge graphs + literature; structured + unstructured | BioDisco |
| Temporal evidence weighting | Recency tracking of supporting evidence | BioDisco |
| Meta-review synthesis | Final integration agent above the debate | AI Co-Scientist |

---

## The Missing Role: The Replicator Agent

No current system implements a dedicated **replication agent** - an agent
whose role is to independently re-run the key experiment from scratch, with
no access to the original agent's reasoning, to check if the result holds.

This is the most important missing component relative to human science.
Replication is the final arbiter of scientific claims. Until multi-agent
systems include genuine replication (independent re-execution, not just
re-evaluation of the same data), they cannot achieve the epistemic standard
that human science (at its best) demands.
